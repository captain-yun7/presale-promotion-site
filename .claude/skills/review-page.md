# 프로젝트 페이지 품질 리뷰

분양 프로젝트 페이지의 전체적인 품질을 점검하고 개선점을 제안합니다.

## 점검 대상 결정

- `target` 인자가 있으면 해당 페이지 점검
- 없으면 사용자에게 질문

## 점검 체크리스트

### 1. 필수 기능 동작
- [ ] 상담 신청 폼 존재 및 필드 완성도 (이름, 연락처, 관심 평형)
- [ ] 폼 제출 → Supabase 저장 로직
- [ ] 폼 제출 → 텔레그램 알림 전송 로직
- [ ] FloatingCTA (플로팅 상담 버튼) 존재
- [ ] 전화번호 `tel:` 링크 동작
- [ ] UTM 파라미터 추적 (`lib/utm.ts` 연동)

### 2. 전환율 최적화 (CRO)
- [ ] CTA 버튼 가시성 및 문구 적절성
- [ ] 상담 신청 폼 접근 용이성 (스크롤 없이 도달 가능?)
- [ ] 긴급성/희소성 요소 (분양 일정, 잔여 세대 등)
- [ ] 신뢰 요소 (시행사/시공사 정보, 인허가 정보)
- [ ] FAQ 섹션으로 의문 해소

### 3. UI/UX 품질
- [ ] 모바일 반응형 (breakpoint: sm, md, lg)
- [ ] 스크롤 애니메이션 동작
- [ ] 이미지 로딩 (placeholder 또는 실제 이미지)
- [ ] 네비게이션 스크롤 이동
- [ ] 폰트 로딩 상태

### 4. 코드 품질
- [ ] TypeScript 타입 안전성
- [ ] 컴포넌트 크기 적정성 (1000줄 이상이면 분리 검토)
- [ ] 중복 코드 여부
- [ ] 하드코딩된 값 관리 (상수 분리)
- [ ] 에러 처리 (API 호출 실패 시)

### 5. 접근성 (A11y)
- [ ] 키보드 네비게이션
- [ ] 색상 대비 (WCAG AA 기준)
- [ ] 폼 라벨 연결
- [ ] 스크린리더 대응

## 자동 수정

사용자 동의 시 아래 항목은 바로 수정:
- 누락된 텔레그램 알림 연동
- FloatingCTA 추가
- UTM 추적 코드 추가
- 기본적인 접근성 개선 (alt, label 등)
